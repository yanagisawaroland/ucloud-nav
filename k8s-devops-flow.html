☸️ Kubernetes + DevOps 逻辑关系与流程

动态 CI/CD 流水线 · 持续交付核心链路
📝 1. 开发阶段
开发者提交代码到 Git 仓库，触发 Webhook。
分支策略: feature → develop → main/release。
Git 版本控制代码评审 PRSonarQube 静态扫描
⬇️
📦 2. 构建 & 单元测试
CI 系统拉取代码，执行 Maven/Gradle 构建，运行单元测试，产出 Jar/二进制文件。
Maven/GradleJUnit/TestNGSpotBugs
⬇️
🐳 3. 镜像构建 & 推送仓库
编写 Dockerfile，构建容器镜像，推送到 Harbor / Docker Hub。
镜像标签: commit hash + 环境标识。
Docker/ContainerdHarbor镜像安全扫描
⬇️
🚀 4. Kubernetes 部署
ArgoCD / Flux 或 kubectl apply 将资源部署到 K8s 集群。
支持蓝绿/金丝雀发布，滚动更新策略。
Helm ChartsKustomizeArgoCD服务网格
⬇️
📊 5. 可观测性 & 运维
Prometheus 采集指标，Grafana 展示仪表盘，Loki 收集日志，Jaeger 链路追踪。
基于 HPA 自动扩缩容，告警通知。
Prometheus+GrafanaLoki/ELKJaeger/KialiVPA/HPA
🔄 逻辑关系
代码变更 → CI 自动构建 → 镜像升级 → CD 触发部署 → 监控反馈 → 优化迭代
⚙️ 动态特性
GitOps 声明式 + 事件驱动流水线 + 自动回滚 + 智能告警

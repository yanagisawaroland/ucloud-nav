Velero 核心概念与实战要点

1. Velero 是什么？

Velero = Kubernetes 的“集群级备份 + 卷快照 + 灾难恢复”工具。

它备份：Deployment / Service / Ingress / CRD / CR / Namespace / RBAC / PV 数据。

它不备份：节点本身、容器镜像、非 K8s 管理的外部资源。

2. 备份原理

资源备份：将 K8s 资源导出为 JSON/YAML 存入对象存储。

卷备份：通过 CSI Snapshot 或 Restic 备份 PV 数据。

3. 常用备份命令

velero backup create full-backup --include-namespaces '*'

velero backup create ns-dev --include-namespaces dev

velero backup create app-nginx --selector app=nginx --default-volumes-to-restic

4. 常用恢复命令

velero restore create --from-backup full-backup

velero restore create restore-dev --from-backup ns-dev --namespace-mappings dev:dev2

velero restore create restore-cross --from-backup full-backup（跨集群恢复）

5. 常见坑与建议

CRD 恢复顺序：先恢复 CRD，再恢复 CR。

StorageClass 不一致时：可使用 --restore-volumes=false，再手动处理 PV。

建议使用 CSI Snapshot 替代 Restic 作为生产环境卷备份方案。

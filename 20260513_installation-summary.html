安装成功总结

devops81 — K3s v1.31.5+k3s1 / Cilium v1.19.1 / Istio Ambient v1.29.0

一、架构层面（最关键）

之前 devops91 ❌本次 devops81 ✅
k3s 缺省模式（不 disable kube-proxy）--disable-kube-proxy
Cilium kubeProxyReplacement=falseCilium kubeProxyReplacement=true
路由走 vxlan tunneleBPF 直通，无 tunnel

→ metrics-server 不被 tunnel 拦截，避免 0/1 问题

二、registries.yaml 配置

  • 协议用 http:// 不用 https://
  • 加 docker.io mirror 指向 Harbor rancher 项目
  • pause、coredns 等基础镜像从本地拉，不依赖外网

三、基础镜像版本匹配

组件devops91 踩坑devops81 成功
corednsHarbor 1.14.2 不匹配 k3s v1.35.4用阿里云拉到正确版本 1.12.0
metrics-server版本不匹配拉到 v0.7.2
local-path版本不匹配拉到 v0.0.30

四、Istio Ambient 安装步骤

  • 提前创建 两个节点 /opt/cni/bin/istio-cni 符号链接
  • excludeNamespaces 必须包含 [kube-system, istio-system]
  • 镜像路径 用 Harbor 正确命名（cilium-cilium 不是 cilium）

五、操作规范

  1. 国产镜像源 rancher-mirror.rancher.cn 避免 GitHub 下载失败
  2. 先配 registries.yaml 再装 Cilium，避免镜像拉不到反复重来
  3. 每一步确认后再继续，不跳步
  4. 外网拉不动时用阿里云 registry.aliyuncs.com/google_containers/ 保底

踩坑记录

K3s v1.31.5+k3s1 / Cilium v1.19.1 / Istio Ambient v1.29.0

问题一：istio-ingressgateway / ztunnel 卡 ContainerCreating

错误 FailedCreatePodSandBox - context deadline exceeded

根因

  • istio-cni 二进制不在 /opt/cni/bin/ 中 → 创建符号链接
  • istio-system 命名空间未被排除 → istio-cni 跳过自己的组件

修复

# 1. 两个节点创建符号链接
mkdir -p /opt/cni/bin && ln -sf /var/lib/rancher/k3s/data/current/bin/istio-cni /opt/cni/bin/istio-cni

# 2. istio-ambient.yaml 必须加
cni:
  excludeNamespaces:
    - kube-system
    - istio-system

问题二：metrics-server 0/1

错误 dial tcp xx:10250: connect: connection refused

根因

Cilium 使用 kubeProxyReplacement=false + vxlan tunnel 时，Istio Ambient 的 SNAT 流量回包被 eBPF 拦截。

解决（架构层面）

--disable-kube-proxy + kubeProxyReplacement=true → eBPF 直通，不走 tunnel

问题三：rancher 基础镜像拉不到

错误 ImagePullBackOff / ErrImagePull

根因

K3s 不同版本需要不同版本的 coredns / local-path-provisioner / metrics-server。v1.31.5 需要旧版本。

修复

# 从阿里云拉取指定版本推送到 Harbor rancher 项目
docker pull registry.aliyuncs.com/google_containers/coredns:1.12.0
docker pull registry.aliyuncs.com/google_containers/metrics-server:v0.7.2
docker pull rancher/local-path-provisioner:v0.0.30
registries.yaml 的 endpoint 用 http:// 不是 https://

关键参数速查

参数值说明
--disable-kube-proxyk3s server让 Cilium eBPF 接管
kubeProxyReplacementtrue配合 disable-kube-proxy
cni.exclusivefalse允许 Istio CNI 链式插入
excludeNamespaces[kube-system, istio-system]防止 istio-cni 跳过自身
registries endpointhttp://不用 https://

Metrics 流： 应用 / Envoy Sidecar → Istio Telemetry / Prometheus Scrape → Prometheus / Thanos → Grafana / Kiali
Logs 流： Envoy Access Log / 应用日志 → Fluent Bit / Promtail → Loki / Elasticsearch → Grafana / Kibana
Traces 流： Envoy Sidecar / 应用 SDK → OpenTelemetry Collector → Tempo / Jaeger / Zipkin → Kiali / Grafana
控制与配置： Istio 控制面 → 下发 Sidecar 配置（路由 / 策略 / Telemetry）→ 影响上述三类数据流
提示：该图仅展示逻辑流向，实际部署中可根据规模拆分为多集群、多 Region、多租户架构。

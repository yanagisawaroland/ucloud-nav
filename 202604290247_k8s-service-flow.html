本图展示 K8s Service 完整流量链路：Client → CoreDNS → ClusterIP → kube-proxy → iptables/IPVS → Endpoint → Pod，涵盖 NodePort/LoadBalancer 外部入口。

Step 1 — Client / Pod：请求发起方

集群内 Pod 或集群外客户端发起请求。集群内访问使用 serviceName.namespace.svc.cluster.local 域名，由 Pod 的 /etc/resolv.conf 指向 CoreDNS 解析。

Step 2 — CoreDNS：服务发现

CoreDNS 监听 UDP/TCP 53，将 Service 域名解析为 ClusterIP（虚拟 IP）。ndots:5 搜索域确保短名称可被补全。Headless Service（clusterIP: None）直接返回 Pod IP 列表。

Step 3 — Service · ClusterIP：虚拟入口

ClusterIP 是仅在集群内可路由的虚拟 IP，由 kube-apiserver 分配，本身没有对应网卡。流量到达 ClusterIP 后由内核 iptables/IPVS 规则进行 DNAT，转发到真实 Pod IP。

Step 4 — kube-proxy：规则同步器

kube-proxy 以 DaemonSet 运行在每个节点，Watch Service 和 Endpoints 变化，将规则写入内核。支持三种模式：userspace（废弃）、iptables（默认）、ipvs（高性能）。

Step 5 — Endpoints / EndpointSlice：后端列表

Endpoints 对象记录所有就绪 Pod 的 IP:Port，由 EndpointController 根据 Pod readinessProbe 动态维护。K8s 1.21+ 默认启用 EndpointSlice，解决大规模集群 Endpoints 对象过大问题。

Step 6 — iptables：内核链路转发

kube-proxy 在 PREROUTING 和 OUTPUT 链插入 DNAT 规则，将目标 ClusterIP:Port 随机替换为某个 PodIP:Port。规则数量随 Service/Pod 数线性增长，大规模场景（>5000 Service）性能下降明显。

Step 7 — IPVS：高性能负载均衡

IPVS 基于内核 LVS 模块，使用哈希表匹配，O(1) 时间复杂度，支持 rr（轮询）、lc（最少连接）、sh（源哈希）等算法。启用方式：kube-proxy --proxy-mode=ipvs，需节点安装 ipvsadm 和加载内核模块。

Step 8 — API Server：控制平面枢纽

kube-proxy、EndpointController、CoreDNS 均通过 Watch 机制订阅 API Server 的 Service/Endpoints 变更事件，形成控制闭环：kubectl apply → API Server → etcd → Watch → 各组件同步。

Step 9-11 — Pod 副本：实际处理流量

后端 Pod 通过 spec.selector 标签被 Service 选中。Pod 就绪（readinessProbe 通过）后加入 Endpoints；Pod 终止或探针失败时从 Endpoints 摘除，实现零中断滚动更新。

Step 12 — NodePort / LoadBalancer：外部入口

NodePort 在每个节点开放 30000-32767 端口，外部流量 → NodeIP:NodePort → ClusterIP → Pod。LoadBalancer 在云厂商创建外部 LB，自动分配公网 IP，底层仍依赖 NodePort 机制。

Step 13 — External Client：集群外访问

外部客户端通过 NodeIP:NodePort 或云 LB 公网 IP 访问集群。externalTrafficPolicy: Local 可保留源 IP（仅转发到本地节点 Pod，避免二次跳转），Cluster 模式则全局均衡但丢失源 IP。

🖥️ Client 访问方式

  • 集群内: svc.ns.svc.cluster.local
  • 集群内: ClusterIP:Port 直连
  • 集群外: NodeIP:NodePort
  • 集群外: LB 公网 IP

🌐 Service 类型

  • ClusterIP（默认，仅内部）
  • NodePort（节点端口暴露）
  • LoadBalancer（云 LB）
  • ExternalName（CNAME 代理）

📋 Endpoints 要点

  • readinessProbe 控制加入/摘除
  • EndpointSlice 分片解决规模问题
  • kubectl get ep 查看后端列表
  • Headless 直接返回 Pod IP

⚙️ kube-proxy 模式

  • iptables: 默认，规则链 O(n)
  • ipvs: 高性能 O(1) 哈希匹配
  • --proxy-mode=ipvs 启用
  • 需加载 ip_vs 内核模块

🔥 iptables 链路

  • PREROUTING → KUBE-SERVICES
  • KUBE-SVC-xxx → KUBE-SEP-xxx
  • statistic 模块随机选择
  • conntrack 保证会话一致性

⚡ IPVS 调度算法

  • rr: 轮询（默认）
  • lc: 最少连接数
  • sh: 源地址哈希
  • ipvsadm -Ln 查看规则

🔵 Pod 健康管理

  • readinessProbe: 流量摘除
  • livenessProbe: 重启容器
  • terminationGracePeriodSeconds
  • preStop hook 优雅退出

🔍 CoreDNS 配置

  • ConfigMap: coredns (kube-system)
  • ndots:5 搜索域补全
  • cache 插件减少上游查询
  • hosts/rewrite 自定义解析

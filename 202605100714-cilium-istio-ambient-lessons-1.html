Cilium + Istio + Ambient Mesh 踩坑总结

🔥 核心问题总结

  • Istiod 多 ReplicaSet 混乱（版本污染）
  • Helm upgrade + rollout 混用导致控制面不收敛
  • Webhook 残留导致 admission 异常
  • RBAC / ServiceAccount / token 状态不一致
  • Readiness probe 一直 503

⚠️ Cilium 常见坑

  • Envoy sidecar 与 eBPF 模式冲突
  • kube-proxy replacement 未统一配置
  • 网络 sandbox 创建失败（CNI未稳定）

⚠️ Istio 常见坑

  • 多版本 istiod 共存（最致命）
  • ConfigMap 访问 Unauthorized
  • Webhook 未同步升级
  • metrics-server 未就绪导致 HPA 误判

⚡ Ambient Mesh 特点

  • 无需 sidecar（ztunnel + waypoint）
  • 更依赖 CNI（如 Cilium）
  • 对 control plane 一致性要求更高

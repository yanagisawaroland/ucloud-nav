流程说明：
1. Client 发送请求到 Service 的 ClusterIP
2. Kube-proxy 监听 API Server，维护转发规则
3. 请求经过 Netfilter（iptables 或 IPVS）进行 DNAT 转换
4. Endpoints 控制器维护 Service 与 Pod IP 的映射关系
5. 数据包被转发到后端 Pod 实际处理请求

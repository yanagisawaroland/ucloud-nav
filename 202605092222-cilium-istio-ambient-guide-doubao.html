Cilium
eBPF-powered Kubernetes Networking & Security
🧬 核心定位
Cilium 是基于 eBPF（扩展伯克利包过滤器） 的 Kubernetes 网络插件（CNI），在 Linux 内核层面实现高性能网络策略、负载均衡和可观测性，无需 kube-proxy，延迟极低。
⚡ 核心组件
  • cilium-agent：每节点守护进程，管理 eBPF 程序
  • cilium-operator：集群级控制器，管理 IPAM/KVStore
  • Hubble：内建可观测平台（流量可视化 + Prometheus 指标）
  • Hubble UI：服务依赖图形界面
  • Hubble Relay：多节点流量聚合
🛡️ 功能矩阵
L3/L4 网络策略
L7 HTTP/gRPC 策略
替代 kube-proxy
Hubble 可观测性
Egress Gateway
🎯 适用场景
  • 追求极低延迟的高吞吐微服务集群
  • 需要细粒度网络安全策略（零信任）
  • 替换 kube-proxy 提升 Service 性能
  • 需要网络可观测性但不想引入额外 sidecar
  • 多集群 ClusterMesh 互联场景

Cilium 安装要点
前置条件 · Helm 安装 · 注意事项
⚠️ 内核版本：要求 Linux 内核 ≥ 4.9.17，推荐 ≥ 5.10（XDP/WireGuard 完整支持）。Cilium v1.14+ 推荐内核 ≥ 5.15。
⚠️ 移除 kube-proxy：若使用 Cilium 替换 kube-proxy，初始化时加 --skip-phases=addon/kube-proxy，已运行集群须先删除 kube-proxy DaemonSet。
⚠️ CNI 冲突：严禁与 Flannel/Calico 同时运行，迁移前需彻底清理旧 CNI 的 iptables 规则和网卡。
✅ 安装顺序：先安装 Helm chart，等待所有 cilium-agent pod Running，再运行 cilium status 验证。
📦 Helm 安装命令
# 添加 repo
helm repo add cilium https://helm.cilium.io/
helm repo update

# 安装（替换 kube-proxy 模式）
helm install cilium cilium/cilium \
  --version 1.15.6 \
  --namespace kube-system \
  --set kubeProxyReplacement=true \
  --set k8sServiceHost=MASTER_IP \
  --set k8sServicePort=6443 \
  --set hubble.relay.enabled=true \
  --set hubble.ui.enabled=true

# 验证
cilium status --wait

Istio Ambient Mesh
Sidecar-less Service Mesh — ztunnel + waypoint proxy
🧩 核心定位
Istio Ambient 是 2022 年推出的无 Sidecar 服务网格方案。将数据面拆分为两层：ztunnel（节点级 L4 mTLS 隧道）和Waypoint Proxy（按需部署的 L7 Envoy），彻底解决 Sidecar 模式的资源开销和侵入性问题。
🏗️ 核心组件
  • istiod：控制面，下发 xDS 配置 + 证书管理
  • ztunnel：每节点 DaemonSet，L4 mTLS 加密隧道（HBONE 协议）
  • Waypoint Proxy：Envoy，按 namespace/SA 粒度部署，处理 L7 策略
  • istio-cni：流量拦截（替代 init container iptables）
  • Kiali：服务拓扑可视化（Ambient 模式完整支持）

🎯 适用场景
  • 不想修改 Pod spec（无 sidecar 注入）
  • 需要 mTLS 加密但资源敏感的环境
  • 大规模集群（Sidecar 数量过多）
  • 仅需 L4 策略时按需启用 Waypoint
  • 与 Cilium CNI 搭配（Cilium + Istio Ambient 最佳实践）

Istio Ambient 安装要点
istioctl / Helm · 前置条件 · 注意事项
⚠️ Kubernetes 版本：要求 K8s ≥ 1.24，推荐 ≥ 1.27（Ambient GA 为 Istio 1.22+）。
⚠️ CNI 兼容性：istio-cni 与某些 CNI 存在 iptables 冲突。Cilium + Ambient 搭配时需开启 --set cni.exclusive=false。
⚠️ Waypoint 按需部署：未部署 Waypoint 时仅有 L4 mTLS，L7 策略（VirtualService/AuthorizationPolicy HTTP）不生效。
✅ 命名空间标签：加入 Ambient 只需 kubectl label ns foo istio.io/dataplane-mode=ambient，无需重启 Pod。
📦 istioctl 安装 Ambient
# 下载 istioctl
curl -L https://istio.io/downloadIstio | \
  ISTIO_VERSION=1.22.2 sh -
export PATH=$PWD/istio-1.22.2/bin:$PATH

# 安装 Ambient 模式
istioctl install --set profile=ambient -y

# 标记 namespace 加入 mesh
kubectl label ns default \
  istio.io/dataplane-mode=ambient

# 部署 Waypoint（L7 策略）
istioctl waypoint apply \
  --enroll-namespace -n default

# 验证
istioctl verify-install

关键安装注意事项
生产环境 Checklist
🐝 Cilium 注意事项
  • 内核 ≥ 4.9（推荐 5.10+），Ubuntu 22.04/24.04 开箱即用
  • 迁移旧 CNI 必须清理残留 iptables：iptables -F; iptables -X
  • kubeProxyReplacement 模式下，master 节点也需运行 cilium-agent
  • Hubble UI 默认 ClusterIP，暴露需 NodePort 或 Ingress
  • 节点重启后 BPF map 会重建，需确认 cilium-agent 先于应用 Pod 启动
  • Calico → Cilium 迁移：停 Calico → 删 CNI config → 安装 Cilium → 重启所有 Pod
🌊 Istio Ambient 注意事项
  • ztunnel 使用 HBONE，需确保节点间 15008 端口可通
  • Waypoint proxy 按 namespace/ServiceAccount 粒度，不是全局的
  • 从 Sidecar 迁移：先 label namespace 再逐步移除 sidecar 注入标签
  • istio.io/use-waypoint annotation 需显式指定，否则 L7 策略不生效
  • 与 Cilium 共存：必须设置 --set cni.exclusive=false
  • Istio 1.22+ GA，生产环境避免使用 1.21 以下 Ambient alpha 版本
🔑 istioctl 通用注意事项
  • 版本严格对应：istioctl 版本必须与 istiod 版本一致
  • 升级前务必运行 istioctl x precheck 预检
  • istioctl analyze 可检测 VirtualService/DestinationRule 配置冲突
  • Ambient 模式下查询 ztunnel：istioctl ztunnel-config workloads
